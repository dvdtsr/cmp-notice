Question 4: using webviews in app

Hello team,

I’m setting your SDK on my Android app but some pages of my checkout funnel are webviews, how can I make sure the banner isn’t displayed twice?

Thanks a lot!

Hello,

To avoid the duplication of the banner display, the idea is to pass the consent information into the webview so that the user does not have to give consent again in each webview.

You will need to:

- integrate the native Didomi SDK into your app to collect consents.
- integrate the web Didomi SDK into the webviews to receive and process the consents collected.
- connect the dots between these two

Here is the related documentation:
https://developers.didomi.io/cmp/mobile-sdk/share-consent-with-webviews#webviews-android-and-ios

Please don't hesitate to contact us if you need more details.

Regards,

Step 2 - Validate a CMP deployment

Our clients usually configure their CMP through the Didomi Console. Once their setup is done, Didomi often double-checks their configuration and confirm that the setup is correct.
If you were tasked with validating a CMP deployment for a client, how would you validate the following elements:

  • The notice tag is correctly deployed
  • The notice behaves as expected
  • The following specific integrations are correctly configured: Google DFP/Ad Manager, Prebid, Google Tag Manager

We expect you to present your findings and thought process during one of your interviews. No written answer is expected for this step. We understand that you might not be familiar with some of the tools listed (DFP, GTM, etc.), in which case you can ignore that part or do only one of them.

I would like to list how I proceed and I hope that we can discuss this later.

Is the notice tag correctly deployed?

Layout

  • I visited the page Le Parisien in incognito mode to make sure the notice shows up.
  • verified if the consent notice is deployed (visually).
  • checked the style: as the notice is directly integrated onto the page, the notice's CSS rules could inherit unexpected CSS declarations from the publisher.

Behavior

  • I openned the dev tools
  • checked the network tab to verify if the SDK is loaded without any error
  • checked the console and verified if no Javascript error related to the API ocurrs.
  • clicking the "I consent" button in the consent notice should trigger the storage of additional cookies.

Are the following specific integrations correctly configured: Google DFP/Ad Manager, Prebid, Google Tag Manager?

Google DFP/ Ad Manager

The following documentation provides details on Google integration:

https://developers.didomi.io/cmp/web-sdk/third-parties/direct-integrations/google-dfp-adsense-adx/share-consent-and-refresh

On the publisher’s page run the following JS:

(Didomi.getConfig()).app.vendors

The result should include: The Google IAB vendors (here, all: true) & "Google" in the Didomi vendor's list

Prebid

Through the global prebid object “pbjs” exposed on the page, we can verify if the CMP is registered in prebid.

https://developers.didomi.io/cmp/web-sdk/third-parties/direct-integrations/prebid#configure-prebid

On the publisher’s page run the following JS:

(pbjs.getConfig()).consentManagement

The CMP is registered.

The following prebidjs documentation provides more details on the possibles options :

https://docs.prebid.org/dev-docs/modules/consentManagement.html#page-integration

GTM
